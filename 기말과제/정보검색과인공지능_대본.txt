✅ Slide 1. 제목 / 팀 소개 — 발표 대본

안녕하세요, 정보검색과 인공지능 발표를 맡은 인공지능융합공학부 202404178 강우현입니다.
저는 카카오맵(Kakao Map) 을 주제로 조사·분석을 진행했습니다.

오늘 발표는 총 10가지 목차로 구성되어 있으며,
카카오맵의 개요부터 데이터 구조, 작동 원리, AI 확장까지 설명드리겠습니다.

🔹 Slide 2. 카카오맵 개요

카카오맵은 카카오에서 제공하는 위치 기반 지도/검색 서비스입니다.
장소 검색, 길찾기, 실시간 교통량을 제공하고, 웹과 앱 모두 이용할 수 있습니다.
특히 카카오톡, 카카오내비, 카카오비즈니스와 연동되어 사용자가 많고 접근성이 좋습니다.

📌 예상 질문

❓ 카카오맵과 네이버지도는 어떤 차이가 있나요?

👉 예상 답변

네이버 지도는 사용자 리뷰와 장소 데이터가 강점이고,
카카오맵은 카카오톡 공유와 실시간 교통 데이터가 강점입니다.

🔹 Slide 3. 발전 과정 (연도별)

카카오맵은 아래와 같은 순서로 발전해왔습니다.

연도	발전 내용
2016.06	다음 지도(DAUM Map)에서 카카오맵으로 전환·출시
2017	카카오내비 기능 연동 (지도 → 내비 바로 길찾기)
2018~2020	카카오톡 공유 최적화 / 장소(POI) 데이터 확장
2022	AI 기반 현재 최적 경로 추천 기능 적용
2024~현재	AI 기반 장소 추천 / AR 길안내 기능 실험 중

📌 예상 질문

❓ 카카오맵이 단순 지도 서비스가 아니라 “정보검색 시스템”이라고 한 이유는?

👉 예상 답변

검색어 입력 → 색인된 데이터 → 결과 제공
이 구조는 지도 앱이 아니라 검색엔진의 동작 방식과 동일하기 때문입니다.

🔹 Slide 4. 그룹별 사용 목적

일반 사용자

장소 검색(카페, 맛집 등), 리뷰 확인, 길찾기

운전자

실시간 교통 → 최적 경로 선택, 카카오내비 연동

소상공인

자신의 매장 정보를 등록하여 노출 및 홍보

📌 예상 질문

❓ 매장 정보는 누가 입력하나요? 카카오가 직접 수집하나요?

👉 예상 답변

카카오가 기본 데이터를 제공하지만,
소상공인이 직접 매장 정보를 등록하거나 수정할 수도 있습니다.

🔹 Slide 5. 작동 방식 (사용자 검색 패턴)

사용자는 명확한 목적을 가지고 지도에서 검색을 진행합니다.

“가까운 곳”

“리뷰 많은 곳”

“현재 영업 중인 곳”

→ 카카오맵은 검색 의도에 따라 필터 / 정렬 기능을 제공합니다.

🔹 Slide 6. 데이터 구성요소

카카오맵 데이터는 3가지 핵심 데이터로 구성됩니다.

데이터 구성요소	설명
지도 데이터 (Map DB)	도로, 건물, 주소, 행정구역
POI 데이터 (Point Of Interest)	장소 정보(DB) → 매장 이름, 영업시간, 메뉴, 전화번호
UGC 데이터 (User Generated Contents)	사용자 리뷰, 평점, 사진
✅ (중요 용어 설명 - 발표 중 간단하게 언급)

POI(Point Of Interest) 는 ‘관심 지점'이라는 뜻으로,
지도에서 검색 가능한 장소 데이터를 의미합니다.
예) 카페, 식당, 편의점, 은행, 병원 등이 POI입니다.

📌 교수님이 많이 질문하는 부분

❓ POI와 UGC의 차이가 무엇인가요?

👉 예상 답변

POI는 카카오가 보유한 공식 장소 데이터,
UGC는 사용자가 남긴 리뷰·사진·평점 데이터입니다.

🔹 Slide 7. 작동 원리 (색인 / 검색 엔진 구조)

카카오맵은 지도 앱이면서 동시에 검색엔진입니다.

검색 흐름은 아래와 같습니다:

사용자가 장소를 입력

색인된 DB(POI + UGC) 에서 검색

연관도 / 거리 / 리뷰 수 / 영업 여부 기준으로 정렬하여 출력

장소 이름, 업종, 메뉴 키워드, 위치 데이터 등이 인덱스로 저장됩니다.
그래서 검색 결과가 빠르고 정확합니다.

📌 교수님이 질문할 가능성이 높은 부분

❓ “색인(Indexing)”이 뭔가요? 검색이랑 뭐가 다른가요?

👉 답변

‘색인’은 검색 속도를 높이기 위해
데이터를 찾기 좋은 형태로 미리 정리해두는 작업입니다.
책으로 치면 “목차(인덱스) 만들기” 와 같은 개념입니다.

🔹 Slide 12. 마무리

카카오맵은 지도 앱이 아니라 검색엔진 + 데이터 플랫폼입니다.
앞으로 AI가 결합되면 예측 기반 길찾기 → 추천 서비스 로 발전할 것입니다.

✅ 요약 — 교수님 질문 대비 키워드 정리
키워드	반드시 말해야 하는 포인트
POI	검색해서 나오는 ‘장소 데이터’
UGC	리뷰/평점 등 사용자 생성 데이터
색인(Indexing)	데이터를 빠르게 찾기 위해 미리 정리하는 과정
카카오맵의 본질	지도 + 검색엔진 + AI 추천 시스템

UI가 직관적이고 사용성이 좋음

카카오톡 공유 → 카카오내비 바로 길찾기 가능

실시간 교통 정보를 정확하게 반영

로드뷰·스카이뷰 등 다양한 시각 기능 제공

단점

해외 지도 데이터 부족

리뷰/장소 데이터는 네이버 지도보다 적음

업데이트 반영 속도가 늦을 때가 있음

✅ Slide 11. 시행착오 및 해결방안 — 발표 대본
시행착오 문제	해결방안

→ 즉, 문제를 사용자가 직접 신고하고, 시스템이 자동으로 개선하도록 설계됨.

✅ Slide 12. 마무리 — 발표 대본

카카오맵은 지도 + 검색엔진 + AI 추천 시스템이 결합된 서비스입니다.

단순 위치 제공이 아닌,
사용자의 의도를 이해하고 데이터를 기반으로 추천하는 플랫폼으로 성장하고 있습니다.
